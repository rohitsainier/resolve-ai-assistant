Phase 6 just shipped — the agent is now safe and persistent. Cross-session memory means "like last time" works, pinned facts survive forever, and the agent can now propose multi-step plans for you to approve before anything runs. 27 tools total.

✅ What works today (shipped)

🎤 Transcription done

  • Whisper local transcription of the active timeline
  • Word-level timestamps (enables filler detection)
  • Smart cache — invalidates on clip reorder / trim
  • Live progress bar with heartbeat

🟢 Smart markers done

  • 🟢 Highlights (key moments)
  • 🔴 Dead air (long pauses)
  • 🔵 Short-clip candidates (60–90s)
  • 🟡 Chapters (YouTube-style)
  • Preview & approve each before applying

🪶 Filler detection done

  • Finds every um / uh / like / you know
  • Word-level precision (±50ms padding)
  • Shows up as red markers or fed into rough cut

✂ Rough cut done

  • New timeline with dead-air regions removed
  • Rebuilds from original source media
  • Keep regions inverted from DEAD_AIR markers

✂ Shorts timeline done

  • New timeline containing just SHORT_CLIP regions
  • Multi-clip spans handled correctly
  • Concatenated in timeline order

📚 YouTube package done

  • Auto chapter markers on timeline
  • Description file (hook + chapters + hashtags)
  • .srt & .vtt subtitle export

💬 Agent prompt editing done

  • Multi-turn tool use — searches transcript, inspects markers, refines
  • "Find the 3 funniest moments" → search → mark → summarize
  • "Tighten this up" → list dead air → build rough cut
  • Undo support — revert any agent action

🗑 Marker management done

  • Clear ALL markers
  • Clear by specific color
  • 15 Resolve colors supported

🔄 Provider choice done

  • Anthropic Claude or OpenAI
  • Auto-detect by which key is set
  • AI_PROVIDER to force one

👁 Vision analysis done

  • analyze_frame — describe any timeline moment
  • Frame extraction via ffmpeg (seek-accurate)
  • Both Claude & GPT-4o vision endpoints

🖼 Thumbnail picker done

  • suggest_thumbnails — scored by vision model
  • Auto-uses highlight markers if no timestamps given
  • Returns ranked with strengths/weaknesses

🗣 Speaker identification done

  • identify_speakers — LLM clusters from context
  • Segment ranges per speaker
  • Best-effort; not acoustic diarization

🎞 B-roll suggestions done

  • suggest_broll — proposes insertion points
  • Each with description + rationale
  • Style hint support (cinematic, stock, etc.)

📂 Media pool access done

  • list_media_pool — walks all bins
  • Returns name, duration, path, resolution
  • Foundation for future auto-matching

📦 Delivery presets done

  • YouTube 1080p / 4K, TikTok/Reels, Instagram Square, Proxy
  • render_timeline — queues or starts render
  • render_status — live job state
  • Saves to ~/.resolve-ai-assistant/renders/

🔊 Audio analysis done

  • analyze_audio_levels — EBU R128 LUFS measurement
  • Clipping detection (true peak)
  • Silence region detection
  • Plain-English fix recommendations

👤 Creator profiles done

  • 5 built-ins: YouTube, Shorts, Podcast, Corporate, Default
  • Editable — tone, pacing, target LUFS, filler sensitivity, notes
  • Auto-injected into every agent prompt
  • Saved to ~/.resolve-ai-assistant/profiles/

🗂 Batch render shorts done

  • batch_render_shorts — one file per SHORT_CLIP marker
  • Auto-numbered filenames
  • Uses any platform preset (TikTok default)

🎚 Audio normalization done

  • normalize_audio_render — ffmpeg loudnorm
  • Hits any target LUFS (default -14 for YouTube)
  • Produces a standalone normalized WAV

🧠 Cross-session memory done

  • Per-timeline journal auto-saved every run
  • remember / forget pin durable facts
  • recall searches past sessions
  • Recent history auto-injected into agent prompts

🛡 Preview-before-commit done

  • submit_plan asks for user approval
  • UI modal shows every step + args + reason
  • Reject cleanly without any timeline change
